# Excel-to-Fabric Workflow Bridge

**Status:** Not on website (available for future use)
**Category:** Change Management / Adoption

---

## Summary

Built a system allowing users to continue working in Excel locally while data automatically syncs to Fabric. Excel becomes the familiar interface for data entry, while Fabric provides version control, data validation, and feeds downstream reporting.

---

## Challenge

Finance team needed to move away from "Excel hell" — emailed spreadsheets, version control chaos (`final_v3_REAL_final.xlsx`), manual consolidation — but completely eliminating Excel would break their workflow and face massive resistance.

They needed a bridge solution, not a rip-and-replace.

---

## Solution

Built a system where:

- Users continue working in Excel locally (familiar interface)
- Data automatically syncs to Fabric when saved
- Fabric provides version control, validation, and structured storage
- Downstream dashboards and reports pull from Fabric, not from emailed Excel files

**Systems:** Microsoft Fabric, Excel, Power Automate, SharePoint integration, Python data validation

---

## Results

- Smooth transition from spreadsheet-based to structured data architecture
- High finance team adoption because they kept familiar tools
- Eliminated email attachments and version control chaos
- Data now flows automatically from Excel to dashboards without manual exports
